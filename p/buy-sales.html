 ARM 实例的与我们熟悉的型x86-64 架构不同，Graviton 2支持ARM V8.2架构扩展，支持用于原子操作的LSE (Large System Extension) 指令集的扩展，可以提高大系统之间的锁定和同步性能，它还支持FP16和用于机器学习的INT8等，毫不夸张的说，这一次Graviton2给我们带来的足够的惊喜。

AWS Graviton2 处理器支持 Amazon EC2 的T4g，M6g、C6g 和 R6g 实例,其中：

T4g实例是采用arm处理器的新一代低成本突发实例类型。

M6g实例用于具有CPU、内存和网络资源平衡的通用工作负载。

C6g实例用于计算优化的工作负载，例如视频编码、建模和游戏服务器。

R6g实例用于内存优化的工作负载，可处理内存中的大型数据集（如数据库）。

Arm处理器与当前这一代基于 x86 的实例相比，这些Arm实例为各种工作负载（包括应用程序服务器、微服务、高性能计算、电子设计自动化、游戏、开源数据库和内存中的缓存）提供高达 40% 的性价比提升。

AWS Graviton2 处理器也为视频编码工作负载提供增强的性能，为压缩工作负载提供硬件加速，并为基于 CPU 的机器学习推理提供支持。

全新的 EC2 T4g 实例由 AWS Graviton2 提供支持的突发性能。我们推出了 T4g 实例，这是一种由 AWS Graviton2 提供支持的新一代低成本突发实例类型，也是由 AWS 使用 64 位 Arm Neoverse 内核构建的自定义处理器。使用 T4g 实例，您可以以比 T3 实例低 20% 的成本享受高达 40% 的性能优势，从而为更广泛的工作负载提供最佳性价比。

T4g 实例专为大多数情况下不全部使用 CPU 的应用程序而设计，使用与默认启用无限模式的 T3 实例相同的信用模式。 仅在处理繁重数据时才需要高 CPU 性能的生产工作负载示例包括 Web/应用程序服务器、中小型数据存储和许多微服务器。与前几代相比，T4g 实例的性能可以迁移额外的工作负载，如缓存服务器、搜索引擎索引和电子商务平台。
